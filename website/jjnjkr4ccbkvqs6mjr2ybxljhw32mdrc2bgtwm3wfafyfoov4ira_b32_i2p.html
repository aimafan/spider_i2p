Dendrite Pinecone I2P Port

NOT MAINTAINED ANYMORE - NO PUBLIC INTEREST

I am keeping everything around and will reactivate it once there is interest.

TL;DR:
- Make sure "SAM application bride" is enabled and started in your I2P Router Config.
- Download a Dendrite Build and a run-file.
- Run it.
- Point your matrix client (Nheko, Element, ...) at "http://localhost:8008" and register.
- It should connect and discover other peers through the default bootstrap b32.i2p.
- To add new peers, open the run-file and add it to the list comma separated.
- for example "-peer aaa.b32.i2p,bbb.b32.i2p,ccc.b32.i2p"


Development happens over on Github.
Unlike the default monolyth deployment, this utilises dendrites made-for-it pinecone router. I have made some changes to pinecone to allow a custom dialer and adapted the pinecone-demo code to use onramp for setting up the connection over sam. Every peer has a b32.i2p address and it can be used as static peer. You can add multiple of these delimited by comma(,) in the run-file. This also means you can make your secret network of peers, but you should not rely on that for safety, make invite-only e2ee encrypted rooms instead.
The status of this project is Maintained! See the Readme on Github for more information.
