Introduction

In case you didn't figure it out from the name, this is a Minecraft server (version 1.19.2, non-premium aka cracked) on I2P. It's running on a potato so don't expect much performance. Get ready to timeout every now and then and, if you are brave enough, try some fighting with monsters.

Rules

Basically the same as on 2B2T: no generally illegal stuff in any form and no lagging the server on purpose.

IRC

You can join it on Irc2P. Instructions for setting up a tunnel and some client suggestions can be found at i2p-projekt.i2p. The channel name is #i2pcraft.

How to join?

Software

You'll need a "cracked" client or a hacked client with an alt manager. Anything that lets you connect to non-premium servers is fine. My recommendation is MultiMC or a fork of it (except for PolyMC) with some open source hacked client.

Telemetry

This version of Minecraft has forced telemetry, which includes the following:
  • launcher identifier
  • user identifier/XUID
  • client session id
  • world session id
  • game version
  • operating system name and version
  • Java runtime version
  • client and server modded status
  • server type (single player, Realms, custom server...)
  • game mode
I don't know how much this affects privacy since we're not using a premium account or server, but it's better to disable it just in case. Some options that I know of are this mod for fabric and some hacked clients. I haven't created either of those, but don't just blindly trust me.

Alternatively you can use something like OpenSnitch to prevent Minecraft and its launcher from connecting anywhere. If you use it, make sure to not accidentally block your java I2P router ;)

I don't know how much changing tunnel count will change anything, but adding one or two extra might save you from getting kicked. Decreasing tunnel lengths will reduce lag, but it will also weaken your anonymity. The example configuration files are using a tunnel length of 1, which seems to work fine. It could be a good idea to make tunnels close automatically when they're unused to not create unnecessary strain to the network.

In game

Add a new server and set the address as localhost:25565 (replace the port if you set it to something else in your configuration). The server's render- and simulation distance is set to 6, but you can lower them for less lag.
