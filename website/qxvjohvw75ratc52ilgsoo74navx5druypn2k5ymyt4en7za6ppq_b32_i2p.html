How to stream for free in i2p network

1. Download the following software from their official pages.
-i2p/i2pd software for connecting the i2p network
Install and modify the settings as you prefer
-MediaMTX simple media server
-FFmpeg for converting/feeding the stream for the media server

MediaMTX and FFmpeg are binaries only and don't need installation



2. Modify MediaMTX ports the way you want (it creates the conf file on first boot)
Make notions of the port you are going to feed the stream in, since you need it for FFMpeg.
Also make notion of the HLS port, that one is needed later for i2p.
In my example MediaMTX RTSP port is 8367 and HLS port 8568.
OPTIONAL: You can disable WebRTC if you fear IP leaks and all other features unwanted.



3. Configure FFMpeg to feed the stream for MediaMTX server, in my example we are using RTSP stream
Any other software, that is capable of streaming audio/video to server will do just fine as well.

ffmpeg -re -rtsp_transport tcp -i "rtsp://admin:@192.168.1.112" -fps_mode passthrough -vcodec copy -acodec copy -f rtsp "rtsp://localhost:8367/mystream"

With this line it re-streams rtsp stream from rtsp source, that has username, but no password.
Video, audio and FPS are passed through without encoding, in this example.
The output is the MediaMTX localhost address, where you need to add the port you used for RTSP.
In my example I keep the stream the same as source.
The /mystream is important in the end of the line, it is the identifier for the stream you need for the browser as well.
You can chose whatever word you want, but in this example it's /mystream



4. Configure i2p to use HLS protocol/port through http server.
- Open hidden services manager in i2p configuration
- Choose New hidden service -> HTTP and press create


(open the image in new tab, if you want to see it in full-size)


- Enter name and description as you wish
- OPTIONAL: If you keep the stream always on, you can tick 'Automatically start tunnel when router starts'
- The port number is the one you chose for HLS in MediaMTX, which is 8568 in my example.
- Choose Tunnel Performance Options -> and select interactive connection from drop-down menu.
- Otherwise the default settings will work already.
- You can modify the connection settings later,
if your stream gets more popular and you need to allow more connections.

- After everything is correct click save from the bottom of the page.
- Start the server we just created and copy the hostname.



5. Copy the hostname to the browser address line and add the /mystream to the end of the address.
Press enter and in a moment your stream should open, if you have done everything accordingly.
