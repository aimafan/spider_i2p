I2P Router Console im Netzwerk ereichbach machen

Um die Router Console auch über einen anderen Rechner im Netzwerk zu erreichen müsst ihr in der clients.config folgenden Eintrag ändern.

clientApp.0.args=7657 127.0.0.1 ./webapps/ bzw. clientApp.0.args=7657 ::1,127.0.0.1 ./webapps/ in

clientApp.0.args=7657 127.0.0.1,192.168.0.1 ./webapps/
127.0.0.1 ist Euer localer Rechner quasi localhost und 192.168.0.1 ist in diesem Fall die IP Adresse des Netzwerkadapter (kann auch anders sein)

Die clients.config befindet sich in

Linux: \home\"Benutzername"\I2P

Windows 7: C:\Benutzer\"Benutzername"\AppDate\Roaming\I2P

Jetzt müsst Ihr nur noch die IP Adresse des Server in eurem Web Browser eingeben, I2P Router neustarten. Fertig!

Als Beispiel: http://192.168.0.1:7657/

Ihr solltet an dieser Stelle nicht vergessen das jeder in Eurem lokalen Subnetz nun auf die Router Console zugreifen kann. Ihr solltet es mit einem Passwort schützen.
Das macht Ihr unter Einstellungne -> Erweitert -> und fügt dort folgenden Text ein consolePassword=0123456789. Wobei 0123456789 das Passwort ist. Der Standart Benutzername ist admin
I2P Router neustarten. Fertig!

I2P HTTP Proxy einrichten

nach oben

Um auch von einem anderen Rechner das I2P Netzerk zu erreichen müsst ihr in der i2ptunnel.config folgenden Eintrag ändern.

tunnel.0.interface=127.0.0.1 in tunnel.0.interface=192.168.x.y

192.168.x.y ist eine Synonym für alle IP Adressen im lokalen Subnetz (z.B 192.168.0.1)
Die i2ptunnel.config befindet sich ebenfals in

Linux: \home\"Benutzername"\I2P

Windows 7: C:\Benutzer\"Benutzername"\AppDate\Roaming\I2P

Jetzt müsst Ihr nur noch die Proxy Einstellung in eurem Web Browser auf die IP und den Port des Homeservers einstellen

I2P Router neustarten. Fertig!

Auf I2P2.de, getI2P.net, I2P2.i2p oder I2P-Projekt.i2p findet Ihr genauere Informationen zu den Einstellungen in Eurem Browser.

Hier ein Beispiel für Firefox.


Um alles Seiten im WWW und I2p Netz zu ereichen, sollte man sich mal PRIVOXY oder FOXYPROXY anschauen. Privoxy ist ein filternder Proxy für das Hypertext Transfer Protocol (HTTP), und für alle gebräuchlich Betriebsysteme verfügbar (Linux, Mac, Windows). FoxyProxy dagegen ist nur ein Addon für Firefox. Wer einen Zweiten Browser für I2P nutzen möchte, kann sich I2PFox mal anschauen (Was eigentlich nur eine Portable Version des Firefox ist mit ein paar Addons).

Kommunikation über I2P

nach oben

Es gibt verschiedene Wege über I2P zu komunizieren.

Susimail (eMail) ist in der I2P Router Console oben Link bei I2P-Dienste unter Email zu finden.

I2P bote (sichere eMail) kann als Plugin in der I2P Router Console nachinstalliert werden. Einfach den Link von echelon.i2p oder tutorials.i2p kopieren und in der I2P Router Console unter Einstellungen -> Klienten -> dann ganz unten Installation von Zusatzprogrammen wieder einfügen und installieren. Aktuelle Version 0.2.6.

I2P Messenger ist ein end-to-end Verschlüsselte Serverlose Kommunikationsanwendung. Die Kommunikation zwischen zwei Personen kann nur statt finden wenn vorher gegenseitig der destination-key ausgetauscht würde. Den key erhaltet Ihr vom I2P Messenger selber.



Um den I2P-Messenger in einem Netzerk auf mehreren Rechnern verwenden zu können muss man vorher in der clients.config

clientApp.1.args=sam.keys 127.0.0.1 in clientApp.1.args=sam.keys 0.0.0.0 ändern.

Was ich beim testen noch herrausgefunden habe ist, dass man die Tunnelbezeichung am besten nicht ändern sollte den sobald Lehrzeichen oder Sonderzeichen enthalten sind bekommt man keine Verbindung zum I2P Router.
jIRCii ist ein auf JAVA basierender IRC Client der als Plugin in der I2P Router Console nachinstalliert werden kann. Einfach den Link von stats.i2p oder tutorials.i2p kopieren und in der I2P Router Console unter Einstellungen -> Klienten -> dann ganz unten Installation von Zusatzprogrammen wieder einfügen und installieren. Aktuelle Version b45-z15 (2010-03-24).

I2P-Bote Multihome/Serverumzug

nach oben

Wenn I2P-Bote erfolgreich installiert und genutzt wurde kommt irgendwann die Zeit das I2P den Rechner wechseln muss, und somit auch die I2P-Bote Identities und das Adressbuch umziehen. Also wo Finden wir die Identities und das Adressbuch.

Windows 7 C:/User/"Benutzername"/AppData/Roaming/I2P/i2pbote/
Linux home/"Benutzername"/.i2p/i2pbote/

Jetzt müsst ihr nur noch die Dateien adressBook und identities kopieren, I2P neustarten Fertig

eepget

nach oben

Die Benutzung von EEPGET unter Windows mit der Eingabeaufforderung

cd C:\Programme\i2p

java -cp lib\i2p.jar net.i2p.util.EepGet -n 999 http://tutorials.i2p/dat/eepget_url.bat

(the -n 999 means keep retrying for 999 times)
oder alles in einer Zeile
java -cp "C:\Programme\i2p\lib\i2p.jar" net.i2p.util.EepGet -n 999 http://tutorials.i2p/dat/eepget_url.bat
oder aber Ihr nutzt die eepget_url.bat wo Ihr nur noch den Link Einfügen müsst

P2P im I2P

nach oben

iMule sollte den meisten jetzt nicht unbekannt sein, es ist nichts anderes als eMule bloss das es halt über das I2P Netzt verkehrt. Neben iMule gibt es auch noch I2Phex. Meines erachtens ist das jetzt nicht so der Bringer weil in der I2P Router Console bereits ein gutes P2P Programm integriert ist (I2PSnark).

I2PSnark ist ein in die I2P Router Console integrierter Torrent Client der Torrents im I2P Netz down- bzw. up- loaded. Es gibt natürlich auch noch andere Standalone Programme für Torrents (I2Prufus, Robert, PyBit, I2P-bt) auf die ich aber nicht weiter eingehe.

Torrent Seiten & Tracker. Eine gute Anlaufstelle ist POSTMAN mit seiner Seite & Tracker. Weiter gute Adressen - Tracker sind DIFTRACKER, WELTERDE. Es gibt natürlich auch noch weiter Seiten und Tracker aber die Liste würde niemals Komplett oder Aktuell sein deswegen höre ich an dieser stelle auf.
